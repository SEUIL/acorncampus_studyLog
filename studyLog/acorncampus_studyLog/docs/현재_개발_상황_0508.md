## 코드 리뷰 결과 — 발견된 문제점

### HIGH (배포 전 반드시 수정)

#### 1. `admin/comment/list.jsp` 파일 누락
- **위치**: `AdminController.java:290` — `handleCommentList()`
- **현상**: GET `/admin/comment/list.do` 접근 시 500 Internal Server Error 발생
- **원인**: `req.getRequestDispatcher("/WEB-INF/views/admin/comment/list.jsp").forward(...)` 호출하지만 파일이 없음
- **해결**: admin/comment/list.jsp 생성 필요 (admin/post/list.jsp 구조 참고)

### MEDIUM (기능에 영향 있음, 개선 권장)

#### 3. JSON 수동 문자열 조립 — CommentController, LikeController, ReportController
- **위치**:
  - `CommentController.java:67` — `out.print("{\"status\":\"ok\", \"commentId\":" + commentId + "}")`
  - `LikeController.java:68–74` — `String.format(...)` 으로 JSON 직접 조립
  - `ReportController.java:67` — 동일 패턴
- **문제**: Gson을 쓰지 않아 특수문자(역슬래시, 유니코드 등) 포함 에러 메시지가 JSON을 깨뜨릴 수 있음. `AdminController`는 Gson을 올바르게 사용 중 — 이 세 곳도 Gson으로 통일해야 함.
- **개선 예시**:
  ```java
  // 현재 (CommentController)
  out.print("{\"status\":\"ok\", \"commentId\":" + commentId + "}");

  // 개선 (Gson 사용)
  Map<String, Object> result = new LinkedHashMap<>();
  result.put("status", "ok");
  result.put("commentId", commentId);
  resp.setContentType("application/json; charset=UTF-8");
  resp.getWriter().write(new Gson().toJson(result));
  ```

#### 4. `PostController.handleDetail()` — LikeService를 메서드 내 지역 변수로 생성
- **위치**: `PostController.java:113`
- **문제**: `LikeService likeService = new LikeService();` 를 매 요청마다 생성. 다른 Service는 모두 필드 선언인데 일관성 없음.
- **개선**: 클래스 필드로 이동
  ```java
  // 필드 선언부에 추가
  private final LikeService likeService = new LikeService();
  ```

### LOW (코드 품질, 선택적 개선)

#### 5. UserService — TODO 주석이 완성된 코드 위에 남아있음
- **위치**: `UserService.java:22, 56, 82, 95, 111, 134, 165, 223, 240, 257`
- **문제**: 이미 구현된 메서드에 `// TODO: ...` 주석이 가이드라인 형태로 남아있어 미완성처럼 보임
- **개선**: 주석 제거 또는 `// 구현 완료` 로 변경

#### 6. 불필요한 if-else 단순화 — UserService
- **위치**: `UserService.java:82–87` (checkEmailAvailable), `UserService.java:96–102` (getUserById)
- **문제**: if/else로 null 반환하는 패턴을 단순화 가능
  ```java
  // 현재
  if (userDao.findByEmail(email) == null) { return true; } else { return false; }
  // 개선
  return userDao.findByEmail(email) == null;

  // 현재
  UserDetailDto user = userDao.findById(userId);
  if (user != null) { return user; }
  return null;
  // 개선
  return userDao.findById(userId);
  ```

### INFO (참고용, 수정 필요 없음)

#### 7. AdminController — 댓글 삭제 시 softDelete vs hardDelete
- **위치**: `AdminController.java:305` — `commentService.deleteComment()` 호출
- **내용**: 관리자 댓글 삭제가 소프트 삭제를 사용. 게시글 강제 삭제는 hardDelete인데 댓글은 softDelete. 의도적이라면 OK, 아니라면 `forceDeleteComment()`로 변경 검토.

#### 8. SeriesController — null 체크 없이 getLoginUser() 호출

- **내용**: `/l_check/` 경로라 LoginCheckFilter가 보장하지만, 명시적 null 체크 없어 다른 컨트롤러와 일관성이 없음.

## 전체 아키텍처 흐름 (현재 상태)

```
브라우저 요청
    ↓
EncodingFilter (UTF-8)
    ↓
LoginCheckFilter (/l_check/*, /admin/* 보호)
    ↓
Controller (@WebServlet)
    ├── doGet  → Service 호출 → req.setAttribute → JSP forward
    └── doPost → Service 호출 → redirect (또는 AJAX: JSON 응답)
          ↓
        Service (비즈니스 로직, 입력값 검증)
          ↓
        DAO (PreparedStatement, Oracle SQL)
          ↓
        Oracle 19c (blog 계정, testdb)
```

**세션**: loginUser (UserDto 타입) — 30분 타임아웃  
**AJAX 컨트롤러**: CommentController, LikeController, ReportController, AdminController (ban/unban/delete)
