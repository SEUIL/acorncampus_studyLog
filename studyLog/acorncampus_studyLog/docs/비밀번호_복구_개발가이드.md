## 전체 흐름

```
[로그인 페이지]
  "비밀번호를 잊으셨나요?" 링크 클릭
        ↓
GET /user/pwd-reset/forgot.do     → 이메일 입력 폼 (forgot_password.jsp)
        ↓
POST /user/pwd-reset/forgot.do    → 이메일 발송 처리
        ↓
사용자가 이메일 수신 → 링크 클릭
        ↓
GET /user/pwd-reset/reset.do?token=xxx → 새 비밀번호 입력 폼 (reset_password.jsp)
        ↓
POST /user/pwd-reset/reset.do     → 비밀번호 변경 → /user/login.do 리다이렉트
```

### 2. DB 테이블 생성

Oracle에 접속해 `schema.sql` 하단의 DDL을 실행한다.

```sql
CREATE TABLE password_reset_tokens (
    token      VARCHAR2(64)  PRIMARY KEY,
    user_id    NUMBER        NOT NULL,
    expires_at TIMESTAMP     NOT NULL,
    used_yn    CHAR(1)       DEFAULT 'N' NOT NULL,
    CONSTRAINT fk_prt_user  FOREIGN KEY (user_id) REFERENCES users(user_id),
    CONSTRAINT chk_prt_used CHECK (used_yn IN ('Y', 'N'))
);

CREATE INDEX idx_prt_user_id ON password_reset_tokens(user_id);
```

### 3. PasswordResetService.java 구현

`service/PasswordResetService.java`에는 메서드 시그니처와 주석이 작성되어 있다.  
세 메서드를 아래 순서대로 구현하면 된다.

#### requestReset(String email)

```java
public void requestReset(String email) {
    // 1. 사용자 조회
    UserDetailDto user = userDao.findByEmail(email);

    // 2. 이메일 없어도 조용히 종료 (열거 공격 방지)
    if (user == null) return;

    // 3. 토큰 생성 + 만료 시각 계산
    String token = generateToken();
    Timestamp expiresAt = toTimestamp(LocalDateTime.now().plusMinutes(TOKEN_EXPIRE_MIN));

    // 4. DB 저장
    resetDao.insertToken(user.getUserId(), token, expiresAt);

    // 5. 이메일 발송
    MailUtil.sendPasswordResetEmail(email, token);
}
```

#### validateToken(String token)

```java
public PasswordResetTokenDto validateToken(String token) {
    return resetDao.findValidToken(token); // null이면 만료/사용 완료
}
```

#### resetPassword(String token, String newPassword)

```java
public boolean resetPassword(String token, String newPassword) {
    // 1. 토큰 재확인
    PasswordResetTokenDto dto = validateToken(token);
    if (dto == null) return false;

    // 2. 비밀번호 BCrypt 해싱 후 저장
    String hashed = BCryptUtil.hash(newPassword);
    userDao.updatePassword(dto.userId(), hashed);

    // 3. 토큰 무효화
    resetDao.markUsed(token);

    return true;
}
```

### 4. PasswordResetController.java 구현

#### handleForgotSubmit — POST 이메일 발송

```java
private void handleForgotSubmit(HttpServletRequest req, HttpServletResponse resp)
        throws ServletException, IOException {
    String email = req.getParameter("email");

    if (email == null || email.trim().isEmpty()) {
        req.setAttribute("errorMsg", "이메일을 입력해주세요.");
        req.getRequestDispatcher("/WEB-INF/views/user/forgot_password.jsp").forward(req, resp);
        return;
    }

    try {
        resetService.requestReset(email.trim());
    } catch (Exception e) {
        // 발송 실패해도 사용자에게는 동일한 메시지 표시
    }

    req.setAttribute("successMsg", "입력하신 이메일로 재설정 링크를 보냈습니다.");
    req.getRequestDispatcher("/WEB-INF/views/user/forgot_password.jsp").forward(req, resp);
}
```

#### handleResetForm — GET 새 비밀번호 입력 폼

```java
private void handleResetForm(HttpServletRequest req, HttpServletResponse resp)
        throws ServletException, IOException {
    String token = req.getParameter("token");

    if (token == null || resetService.validateToken(token) == null) {
        req.setAttribute("errorMsg", "유효하지 않거나 만료된 링크입니다.");
        req.getRequestDispatcher("/WEB-INF/views/user/forgot_password.jsp").forward(req, resp);
        return;
    }

    req.setAttribute("token", token);
    req.getRequestDispatcher("/WEB-INF/views/user/reset_password.jsp").forward(req, resp);
}
```

#### handleResetSubmit — POST 비밀번호 변경

```java
private void handleResetSubmit(HttpServletRequest req, HttpServletResponse resp)
        throws ServletException, IOException {
    String token       = req.getParameter("token");
    String newPassword = req.getParameter("newPassword");

    if (token == null || newPassword == null || newPassword.trim().length() < 6) {
        req.setAttribute("errorMsg", "비밀번호는 6자 이상 입력해주세요.");
        req.setAttribute("token", token);
        req.getRequestDispatcher("/WEB-INF/views/user/reset_password.jsp").forward(req, resp);
        return;
    }

    boolean ok = resetService.resetPassword(token, newPassword.trim());

    if (!ok) {
        req.setAttribute("errorMsg", "유효하지 않거나 만료된 링크입니다.");
        req.getRequestDispatcher("/WEB-INF/views/user/forgot_password.jsp").forward(req, resp);
        return;
    }

    resp.sendRedirect(req.getContextPath() + "/user/login.do");
}
```

경로: `src/main/webapp/WEB-INF/views/user/forgot_password.jsp`

경로: `src/main/webapp/WEB-INF/views/user/reset_password.jsp`

- 제출 버튼 `<form action="${pageContext.request.contextPath}/user/pwd-reset/reset.do" method="post">`

## 테스트 방법

1. `mail.properties`에 실제 Gmail 앱 비밀번호 입력
2. Oracle에 `password_reset_tokens` 테이블 생성
3. 서버 재시작
4. `http://localhost:8080/acorncampus_studyLog/user/pwd-reset/forgot.do` 접속
5. 가입된 이메일 입력 → 메일 수신 확인
6. 메일 링크 클릭 → 새 비밀번호 입력 → 로그인 성공 확인
7. 같은 링크 재사용 시 "유효하지 않은 링크" 오류 확인
