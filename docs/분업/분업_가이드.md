## 1. 프로젝트 구조 한눈에 보기

```
브라우저 요청
    ↓
Controller (서블릿, URL 라우팅)
    ↓
Service (비즈니스 로직 ← 팀원들이 채워넣는 곳)
    ↓
DAO (DB 쿼리 ← 이미 완성됨, 손대지 마세요)
    ↓
Oracle DB
```

**DAO는 이미 완성되어 있습니다.** 팀원들이 할 일은 크게 두 가지입니다:
1. **Service의 TODO 채우기** — DAO를 호출하는 비즈니스 로직 작성
2. **HTML → JSP 변환** — 완성된 디자인을 JSP로 변환하고 백엔드 데이터 연결

#### 핵심 로직 설명

**CommentService — 댓글 트리 조립**  
DB에서 조회하면 댓글/대댓글이 한 줄씩 나란히 옵니다.  
이걸 부모-자식 구조로 묶어야 JSP에서 들여쓰기 렌더링이 됩니다.

```java
// flat list → 트리 구조 변환 예시
List<CommentDto> all = commentDao.findByPostId(postId);
List<CommentDto> roots = new ArrayList<>();       // 부모 댓글만
Map<Long, CommentDto> map = new LinkedHashMap<>();

for (CommentDto c : all) map.put(c.getCommentId(), c);
for (CommentDto c : all) {
    if (c.getParentCommentId() == null) {
        roots.add(c);
    } else {
        map.get(c.getParentCommentId()).getReplies().add(c);
    }
}
return roots;
```

**LikeService — 좋아요 상태머신**  
같은 타입 다시 클릭 → 취소 / 다른 타입 클릭 → 변경 / 처음 클릭 → 등록

```java
LikeDto existing = likeDao.findByPostAndUser(postId, userId);
if (existing == null) {
    likeDao.insert(postId, userId, likeType);        // 신규 등록
} else if (existing.getLikeType().equals(likeType)) {
    likeDao.delete(postId, userId);                  // 같은 타입 → 취소
} else {
    likeDao.update(postId, userId, likeType);        // 다른 타입 → 변경
}
```

#### UserService 메서드 설명

각 메서드 안에 `// TODO: ...` 주석이 구현 순서까지 적혀있습니다. 그 순서대로 따라가면 됩니다.

- `login(email, password)` — DB에서 이메일로 조회 → BCrypt 비밀번호 확인 → 정지 여부 확인 → UserDto 반환

- `changePassword(userId, currentPw, newPw)` — 현재 비밀번호 확인 후 변경
- `deleteAccount(userId, password)` — 비밀번호 확인 후 소프트 삭제

## 5. 협업 포인트 — 서로 맞닿는 부분

팀원 간 코드가 맞닿는 부분에서 충돌이 날 수 있습니다. 미리 알아두세요.

### 5-1. post_detail.jsp (현겸) ← PostService 데이터 (철진)

`post_detail.jsp`에는 게시글 본문과 댓글이 함께 표시됩니다.  
게시글 데이터는 **철진의 PostService**, 댓글 데이터는 **현겸의 CommentService**에서 옵니다.  
PostController에서 두 Service를 모두 호출해서 request에 담아줘야 합니다.

```java
// PostController.handleDetail() 안에서
PostDto post = postService.getPostDetail(postId);          // 철진 담당
List<CommentDto> comments = commentService.getComments(postId); // 현겸 담당
req.setAttribute("post", post);
req.setAttribute("comments", comments);
req.getRequestDispatcher("/WEB-INF/views/post/post_detail.jsp").forward(req, resp);
```

→ 철진과 현겸이 이 부분을 같이 맞춰봐야 합니다.

### 5-2. post_write.jsp (현겸) ← SeriesService 데이터 (건희)

게시글 작성 화면에는 "시리즈 선택" 드롭다운이 있습니다.  
이 드롭다운 목록은 **건희의 SeriesService**에서 가져옵니다.  
PostController.handleWriteForm()에서 SeriesService를 호출해서 넘겨줘야 합니다.

```java
// PostController.handleWriteForm() 안에서
List<SeriesDto> seriesList = seriesService.getSeriesByUser(loginUser.getUserId()); // 건희 담당
req.setAttribute("seriesList", seriesList);
```

→ 현겸과 건희가 이 부분을 같이 맞춰봐야 합니다.

### 5-3. header.jsp (철진) — 모든 팀원이 사용

모든 JSP 파일 상단에 들어갑니다. 네비게이션, 로그인 상태 표시가 포함됩니다.  
철진이 먼저 완성해서 develop에 올리면 나머지 팀원들이 include해서 사용합니다.

```jsp
<%@ include file="/WEB-INF/views/common/header.jsp" %>
```

### 5-4. 관리자 페이지 (건희) ← 각 Service 데이터 (다른 팀원)

관리자 대시보드에서 회원 수, 게시글 수, 신고 수를 보여줍니다.  
이 숫자들은 UserService(진희), PostService(철진), ReportService(현겸)에서 옵니다.  
AdminController에서 다른 팀원 Service들을 호출합니다.

```java
// AdminController.handleDashboard() 안에서 (이미 일부 구현됨)
int userCount = userService.countAll();      // 진희 담당 Service
int postCount = postService.countAll();      // 철진 담당 Service
int reportCount = reportService.countPending(); // 현겸 담당 Service
```

## 6. 코드 작성 규칙 (필독)

### DB 쿼리 — PreparedStatement만 사용

```java
// 이렇게 하면 절대 안 됩니다 (SQL Injection 위험)
Statement stmt = conn.createStatement();
stmt.executeQuery("SELECT * FROM users WHERE email = '" + email + "'");

// 이렇게 해야 합니다
PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM users WHERE email = ?");
pstmt.setString(1, email);
```

### 비밀번호 — 반드시 BCrypt 해시

```java
// 저장할 때
String hashed = BCryptUtil.hash(rawPassword);

// 확인할 때
boolean ok = BCryptUtil.check(inputPassword, storedHash);
```

### DB 연결 — 반드시 자원 반납

```java
Connection conn = null;
PreparedStatement pstmt = null;
ResultSet rs = null;
try {
    conn = DBUtil.getConnection();
    // ... 쿼리 실행
} finally {
    DBUtil.close(conn, pstmt, rs);  // 반드시 호출
}
```

### JSP 출력 — XSS 방지

```jsp
<%-- 이렇게 하면 XSS 공격에 취약합니다 --%>
<p>${post.title}</p>

<%-- 이렇게 해야 합니다 --%>
<p><c:out value="${post.title}"/></p>
```

### AJAX JSON 응답

```java
resp.setContentType("application/json; charset=UTF-8");
resp.getWriter().write(new Gson().toJson(resultMap));
```

자세한 예시는 `CLAUDE.md`의 "AJAX JSON 응답 작성법" 섹션 참고

```
   git pull origin develop
   git checkout -b feature/내이름/기능명

   git commit -m "feat: 댓글 서비스 구현"

4. 팀장(스일) 코드 리뷰 후 merge

```

```
docs: 문서 수정
style: 코드 스타일만 변경 (줄바꿈, 공백 등)
```
