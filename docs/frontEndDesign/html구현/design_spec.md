# 🎨 스터디로그 (LOG.ME) 디자인 및 스타일 명세서

본 문서는 스터디로그 프론트엔드 개발의 시각적 일관성을 유지하기 위한 **디자인 토큰(Design Tokens) 및 컴포넌트 가이드라인**입니다. 모든 HTML 페이지는 `global_theme.css` 하나에 의존하며, 본 명세서에 정의된 클래스명을 사용하여 스타일을 적용해야 합니다.

---

## 1. 테마 콘셉트 (Themes)

스터디로그는 개발자 친화적이면서도 깔끔한 두 가지 테마를 지원합니다. HTML `<body>` 태그에 `.dark-theme` 클래스를 추가/제거하는 방식으로 테마가 전환됩니다.

*   **☀️ Himalaya (Light Mode - 기본):** 
    *   차가운 메탈릭(Metallic) 톤과 미니멀리즘(Minimalism). 
    *   여백을 살린 노션(Notion) 스타일의 레이아웃.
*   **🌙 Midnight (Dark Mode):** 
    *   IDE(개발 도구)와 유사한 하이테크(High-tech) 감성. 
    *   장시간 작업해도 눈이 편안한 저대비 바탕에 형광 초록 포인트를 사용.

---

## 2. 타이포그래피 (Typography)

*   **기본 폰트:** `Pretendard`, sans-serif
*   **계층 구조:**
    *   `h1`: 24px, Font Weight 800 (페이지 최상단 대제목)
    *   `h2`: 18px, Font Weight 700 (섹션 제목)
    *   `h3`: 16px, Font Weight 600 (카드 및 리스트 아이템 제목)
    *   `p.text-sub`: 14px (서브 설명, 안내 문구)
    *   `본문(body)`: 14px ~ 16px, Line Height 1.6

---

## 3. 컬러 팔레트 (Color Tokens)

CSS 변수(`var()`)를 사용하여 색상을 관리합니다.

| 구분 | 변수명 | Light (Himalaya) | Dark (Midnight) | 용도 |
| :--- | :--- | :--- | :--- | :--- |
| **배경** | `--bg-body` | `#ECEFF1` (연한 회은색) | `#121212` (매우 어두운 회색) | 전체 페이지 배경, 여백 |
| **카드** | `--bg-card` | `#FAFAFA` (거의 흰색) | `#1E1E1E` (어두운 회색) | 사이드바, 폼, 게시글 카드 배경 |
| **텍스트** | `--text-main` | `#263238` (짙은 청회색) | `#E0E0E0` (밝은 회색) | 제목, 본문 등 주요 텍스트 |
| **서브텍스트**| `--text-sub` | `#78909C` (중간 청회색) | `#9E9E9E` (중간 회색) | 메타 정보, 날짜, 플레이스홀더 |
| **테두리** | `--border-color`| `#CFD8DC` | `#333333` | 구분선, 테이블 테두리, 입력창 |
| **포인트** | `--accent-color`| `#607D8B` (블루그레이)| `#00E676` (네온 그린) | **주요 버튼**, 활성 탭, 강조 링크 |
| **입력창** | `--input-bg` | `#ffffff` | `#121212` | `input`, `select`, `textarea` 배경 |
| **호버** | `--list-hover` | `#ECEFF1` | `#252525` | 버튼이나 리스트에 마우스를 올렸을 때 |

## 4. 핵심 컴포넌트 클래스 가이드

html 작성 시 아래의 클래스명들을 조합하여 사용합니다. (css에 이미 정의되어 있습니다.)

### 🔘 버튼 (Buttons)
*   `.btn`: 버튼의 기본 형태 (여백, 둥근 모서리, 폰트 설정). 항상 다른 클래스와 조합.
*   `.btn-primary`: 색상이 꽉 찬 강조 버튼 (ex: 로그인, 게시글 작성 완료).
*   `.btn-outline`: 테두리만 있는 서브 버튼 (ex: 취소, 테마 변경).
*   `.btn-sm`: 관리자 페이지나 테이블 안에서 쓰이는 작은 버튼.
*   `.icon-btn`: 배경 없이 아이콘만 덩그러니 있는 둥근 버튼 (ex: 설정 톱니바퀴).
*   `.btn-danger`: 삭제나 정지 등 위험한 액션을 뜻하는 붉은색 버튼.

### 📝 입력 폼 (Forms)
*   `.input-control`: 일반 사용자 화면의 로그인, 회원가입 등에 쓰이는 인풋/텍스트에어리어.
*   `.admin-input`, `.admin-select`: 관리자 화면의 컨트롤 바에서 쓰이는 검색창/드롭다운.

### 🍱 레이아웃 및 카드 (Layout & Cards)
*   `.dashboard-wrapper`: 전체 화면을 Flexbox로 묶어주는 최상위 컨테이너.
*   `.sidebar`: 좌측에 고정(sticky)되는 280px 너비의 사이드바 메뉴 영역.
*   `.main-content`: 우측의 실제 콘텐츠가 렌더링되는 가변 영역.
*   `.series-card`: 시리즈나 주요 게시물을 표시하는 깃허브 레포지토리 스타일의 박스 요소.

### 🏷 기타 유틸리티
*   `.tag`: `#Java`, `#Backend` 처럼 작은 태그를 표시할 때 사용.
*   `.breadcrumb`: 화면 상단 좌측의 `Home / 메뉴명` 형태의 뒤로가기(네비게이션) 링크.
